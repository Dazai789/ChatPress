# chatpress-v1 产品定义

## 1. 产品概述

chatpress-v1 是一个轻量级 Markdown 页面发布系统。

它的核心目标是把 Markdown 笔记和学习整理内容转化为结构化的 `Artifact`，并发布为可访问、可分享的 HTML 页面。

## 2. 产品定位

chatpress-v1 位于 Markdown 笔记工具和轻量页面发布工具之间。

传统笔记工具适合记录，但公开分享和结构化发布能力有限。博客系统可以发布内容，但对学习笔记和项目过程记录来说流程偏重。

chatpress-v1 的定位是：

```text
把 Markdown 笔记变成可维护、可分享的知识页面。
```

## 3. 目标用户

主要面向：

- 使用 Markdown 整理学习笔记和项目记录的个人用户。
- 想把本地笔记快速发布成 HTML 页面的人。
- 需要整理技术笔记、调研记录和项目过程的开发者。
- 面向求职展示，希望把学习和项目过程整理成可访问页面的用户。

暂不面向多人协作团队、通用博客平台或内容社区。

## 4. 核心场景

基础场景：

```text
用户输入 Markdown 内容
-> 创建 Artifact
-> 系统根据标题自动生成 slug
-> 系统保存原始内容
-> 系统渲染 HTML
-> 用户设置 draft / published
-> published 内容可通过 /p/{slug} 公开访问
```

后续功能仍然围绕这条链路扩展，优先补齐安全、用户、数据库、缓存和工程化能力。

## 5. 当前产品能力

当前产品已经支持：

- 创建、编辑、删除和查看 Artifact。
- 上传或输入 Markdown 内容。
- 将 Markdown 渲染成安全 HTML。
- 使用 slug 生成公开访问地址。
- 使用 draft / published 控制公开可见性。
- 通过后台页面管理 Artifact。

详细接口见 [API.md](API.md)。
详细数据结构见 [DATA_MODEL.md](DATA_MODEL.md)。

## 6. 当前产品缺口

当前还不是完整产品，主要缺口是：

- 没有真实用户账号体系。
- 没有注册 / 登录 API。
- 没有用户数据隔离。
- slug 删除后是否复用还没有形成明确产品规则。
- 公开页面样式和后台管理体验仍然比较基础。

## 7. 当前用户流程

```text
创建 artifact
-> 输入 title 和 Markdown 内容，或上传 .md 文件
-> 保存
-> 修改状态为 published
-> 打开公开页面 URL
```

公开 URL 示例：

```text
/p/my-java-note
```

## 8. 产品原则

- 先打通完整发布链路，再做复杂导入能力。
- 保留原始内容，不丢失用户输入。
- Artifact 是核心对象，其他功能围绕它扩展。
- 后端逻辑优先清晰可理解。
- 每一步都配测试，避免后面改乱。

## 9. 非目标

以下能力暂不作为当前核心目标：

- 真实 AI 模型调用。
- 自动读取 ChatGPT / Claude / Gemini 网页聊天记录。
- AI 聊天记录解析。
- 复制网页 HTML 后自动清洗成 Markdown。
- 浏览器扩展。
- 多用户协作。
- DOCX / PDF 解析。
- 语义搜索。
- 知识图谱。
- 评论系统。
- MCP server。
- 托管 SaaS。

这些能力可以作为 V2/V3 方向，但当前项目先专注 Markdown Artifact 发布链路。
